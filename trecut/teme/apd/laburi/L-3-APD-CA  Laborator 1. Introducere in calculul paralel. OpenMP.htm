Laborator 1. Introducere in calculul paralel. OpenMP

Calcul paralel

Diferite categorii de aplicatii necesita calcul paralel pentru a asigura: crestere a performantelor (diminuare a timpului de executie), fiabilitate (siguranta in functionare), functionalitati distincte, inerent paralele.

Cateva clase conceptuale de probleme sunt: Paralelismul rezultat, Agenda de activitati, Ansamblul de specialisti. Cu aceste tipuri de probleme vom lucra in cadrul laboratoarelor.

Metode de programare ce se vor studia sunt: Comunicarea prin mesaje, Structuri active de date, Date partajate. Clase de sisteme paralele si distribuite la care vom face referire sunt: SISD, SIMD, MISD, MIMD.

OpenMP

OpenMP este un API (Application Program Interface) pentru programarea pralela folosind o memorie partajata in C/C++ si Fortran, multiplatforma (inclusiv Unix si Windows NT); este un model portabil, scalabil care ofera o interfata simpla si flexibila pentru dezvoltarea aplciatiilor paralele.

Model de programare:

  • OpenMP se bazeaza pe paradigma de programare care presupune existenta mai multor threaduri care acceseaza o memorie comuna, partajata global
  • OpenMP foloseste modelul de executie paralela fork-join


Prezentare generala OpenMP

Deoarece un API ca Pthreads este considerat fiind de nivel scazut, el este predominant utilizat de catre programatorii de sistem in dauna celor de aplicatii. S-a observat de asemenea, ca o clasa semnificativa de aplicatii ar putea sa beneficieze major de pe urma unor constructii de nivel mai inalt (directive), ce vin sa scuteasca utilizatorul/programator, de mecanismele uneori complicate ale lucrului cu threaduri. Mai multe astfel de limbaje bazate pe directive au existat de-a lungul timpului, insa doar de relativ putin timp, efortul de a standardiza aceste operatii s-a concretizat in ceea ce astazi se numeste OpenMP. OpenMP-ul este o un API ce poate fi utilizat in C, C++ si FORTRAN, pentru programarea masinilor cu memorie partajata. Directivele OpenMP ofera suport pentru concurenta, sincronizare si manipularea datelor, eliminand in acelasi timp nevoia de a utiliza mutex-uri, semafoare, variabile conditionale, de scop si de a avea grija de initializarile complicate. Pe scurt, OpenMP-ul este un API compus din directive compilator, rutine de biblioteca runtime, si variabile de context.

Directivele OpenMP sunt bazate (in C/C++) pe directivele de compilator, sau pragma-uri. O directiva este construita din numele directivei, urmat de diverse clauze. OpenMP-ul utilizeaza modelul fork-join al executiei paralele. Cu toate ca acest model poate fi utilizat pentru o multitudine de probleme, el este restrans oarecum aplicatiilor bazate pe lucrul cu vectori (calcul vectorial). Una dintre scopurile funcamentale ale OpenMP-ului este acela de a oferi suport pentru programe care sa se execute corect atat serial cat si paralel. Adica, in cazul in care se doreste o executie seriala, programul se compileaza si ruleaza serial fara nici o problema; pentru utilizarea bibliotecii si paralelizarii OpenMP, se compileaza si se link-eaza corespunzator cu OpenMP-ul, urmand ca rularea sa foloseasca threaduri multiple pentru executia programului in paralel.



Programele paralelizate utilizand API-ul OpenMP, incep cu un singur thread master, ce executa in mod serial toate regiunile programului, pana cand se intalneste prima regiune paralela. Directivele paralele constituie in OpenMP regiuni paralele. In acest moment, thread-ul master creaza un grup de thread-uri, el insusi devenind master-ul lor (cu id-ul 0). Fiecare thread executa apoi in mod dinamic instructiunile din cadrul regiunii paralele (in afara de regiunile de sincronizare/comunicare). Daca un thread modifica un obiect partajat (din memoria partajata), acest lucru afecteaza nu numai propriul mediu de executie, ci si pe cel al celorlalte thread-uri din program (aceasta este de altfel si ideea pentru care sunt folosite aceste constructii). La terminarea regiunii paralele, thread-urile din grup se sincronizeaza printr-o bariera implicita, si singurul care continua apoi executia seriala este thread-ul master. Intr-un program pot exista oricate regiuni paralele. In acest fel, un program poate sa foloseasca aceste regiuni fork-join, in mod repetat, utilizand astfel cat mai bine, resursele puse la dispozitie de sistem.

OpenMP-ul este gandit ca un model de programare explicit (ca si MPI-ul) dar nu automatic, oferind astfel programatorului controlul deplin asupra paralelizarii. O alta caracteristica important oferita de OpenMP este suportul pentru imbricarea constructiilor paralele (nested parallelism) - problema aici este reprezentata de numarul, relativ scazut inca, de implementari functionale ale acestei facilitati. API-ul ofera de asemenea si posibilitatea de a modifica in mod dinamic numarul de threaduri ce se pot executa in regiunile paralele - aceeasi mentiune de mai sus se aplica si aici: nu toate implementarile suporta acest lucru.

Directive OpenMP

Formatul directivelor in C / C++ este: #pragma omp directive-name [clause, ...] newline, unde:

#pragma omp este necesara pentru toate directivele OpenMP in C/C++

directive-name trebuie sa specifice un nume valid de directiva OpenMP. Trebuie sa apara dupa pragma si inainte de eventualele clauze ale unei directive.

[clause, ...] sunt optionale, pot aparea in orice ordine si repetate dupa necesitati.

newline este absolut necesara. Dupa linia noua urmeaza blocul structurat ce este referit de aceasta directiva.

Exemplu: #pragma omp parallel default(shared) private(beta,pi)

Reguli generale ale directivelor:

  • Urmeaza conventiile si standardele compilatoarelor de C si C++
  • Sunt case sensitive
  • Intr-o directiva, doar un singur nume poate fi specificat.
  • Fiecare directiva se aplica doar statement-ului urmatoar, care trebuie sa fie un bloc structurat.
  • Directivele "lungi" pot fi continuate pe liniile succesive prin adaugarea unui \ la sfarsitul primei linii a directivei.


Enumerarea Directivelor OpenMP:

  • PARALLEL region: un bloc ce va fi executat in paralel de OMP_NUM_THREADS thread-uri. Este constructia fundamentala in OpenMP.
  • Structuri Work-sharing:
  • DO/for - partajeaza o iteratie a unui ciclu peste toate thred-urile (data parallelism);
  • SECTIONS - sparge task-ul in sectiuni separate (functional parallelism);
  • SINGLE - serializeaza o portiune de cod.
  • Constructii de sincronizare:
  • MASTER - doar thread-ul master va executa aceasta regiune de cod;
  • CRITICAL - regiunea respectiva de cod va fi executata doar de un singur thread;
  • BARRIER - toate threadurile din pool se sincronizeaza;
  • ATOMIC - o anume regiune de memorie se va updata in mod atomic - un fel de mini sectiune critica;
  • FLUSH - identifica un punct de sincronizare in care memoria trebuie sa fie intr-o stare consistenta;
  • ORDERED - iteratiile ciclului din aceasta directiva vor fi executate in aceeasi ordine ca si in executia seriala corespunzatoare;
  • THREADPRIVATE - este folosit pentru a face din variabile globale, variabile locale separate ce vor fi executate pe mai multe regiuni paralele;


Clauze pentru setarea contextului:

Sunt importante pentru programarea intr-un model de programare cu memorie partajata. Se folosesc impreuna cu directivele PARALLEL, DO/for si SECTIONS.

  • PRIVATE - variabilele din lista sunt private in fiecare thread.
  • SHARED - variabilele din lista sunt partajate intre thread-urile team-ului curent;
  • DEFAULT - permite utilizatorului sa seteze un default "PRIVATE", "SHARED", sau "NONE" pentru toate variabilele dintr-o regiune paralela;
  • FIRSTPRIVATE - combina functionalitatea clauzei PRIVATE cu initializarea automatizata a variabilelor din lista: initializarea variabilelor locale se face cu valoarea anterioara intrarii in ciclu;
  • LASTPRIVATE - combina functionalitatea clauzei PRIVATE cu o copie a ultimei iteratii din sectiunea curenta;
  • COPYIN - ofera posibilitatea de a asigna acceasi valoare variabilelor THREADPRIVATE pentru toate thread-urile din pool;
  • REDUCTION - realizeaza o reducere pe variabilele care apar in lista (cu o operatie specifica: + - * /, etc).

  • Pentru a compila pe statiile de lucru din laboratorul ED120 surse C/C++ cu suport OpenMP: gcc -fopenmp my_program.c
  • Pentru a compila pe Sun Enterprise (laboratorul CoLab) surse C/C++ cu suport OpenMP: cc -xopenmp -xO3 my_program.c
  • Pentru a compila pe fep.grid.pub.ro aceste programe, se poate folosi urmatoarea linie de comanda (gcc): gcc -fopenmp -fdump-tree-all file_name.c -o binary_name
  • Pentru a compila pe fep.grid.pub.ro aceste programe, se poate folosi urmatoarea linie de comanda (SunStudio): cc -xopenmp -xO3 file_name.c -o binary_name
  • Utilizati pentru definirea numarului de threaduri o structura de genul #define NUM_THREADS 2, combinata cu functia omp_set_num_threads(NUM_THREADS); Similar, se poate folosi in linie de comanda export OMP_NUM_THREADS=4 pentru a crea de exemplu 4 threaduri
  • Pentru a ajunge pe fep, utilizati ssh -l username fep.grid.pub.ro -v
  • Pentru a copia fisiere pe fep, utilizati scp file_name username@fep.grid.pub.ro:.
